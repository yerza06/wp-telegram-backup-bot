# Changelog

Все значимые изменения проекта документируются в этом файле.

## [Unreleased]

### Added
- Полный бэкап WordPress: копия каталога сайта + дамп БД (`mysqldump`) + `metadata.json` в архиве `.tar.zst`.
- Восстановление из локального бэкапа (`/restore`, `/restore_<id>`) и из внешнего локального архива (`/restore_path`) с механизмом reserve/rollback каталога WordPress.
- Инлайн-меню управления бэкапами (`/backups`): просмотр карточки, восстановление и безвозвратное удаление.
- Проверка диска (`/disk`) и PNG-график использования места (`/disk_chart`) на matplotlib.
- Очистка кэша WordPress (`/clear_cache`) через WP-CLI (`cache flush`, `transient delete --all`) от имени настроенного пользователя.
- Команды `/start`, `/help`, `/status` с reply-меню.
- Ролевой доступ (`viewer`, `admin`, `superadmin`) через `AccessMiddleware`.
- Плановый бэкап по cron-расписанию (APScheduler) с откладыванием при активной тяжёлой операции и уведомлением админов.
- Защита от параллельного выполнения тяжёлых операций (`OperationService`).
- Хранение истории бэкапов и операций в локальной SQLite (SQLAlchemy 2, async).
- Контроль минимального свободного места перед бэкапом и восстановлением.
- Маскирование секретов (токен бота, пароль БД) в логах и сообщениях об ошибках.
- Логирование с ротацией файла `logs/bot.log` и выводом в консоль.
- Пример systemd-юнита для деплоя (`deploy/wp-telegram-backup-bot.service.example`).
