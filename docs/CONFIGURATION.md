# Конфигурация

Настройки читаются из `.env` через `pydantic-settings` (`bot/core/config.py`). Вложенные группы используют разделитель `__`: например `TELEGRAM__BOT_TOKEN`. Шаблон со всеми переменными — в `.env.example`.

Группы `telegram`, `wordpress`, `backup` обязательны (у них есть обязательные поля без значений по умолчанию). Группы `core`, `disk_chart`, `schedule`, `tools` имеют значения по умолчанию и могут не указываться.

## Telegram (`TELEGRAM__*`)

| Переменная | Тип | Обязательно | По умолчанию | Описание |
|------------|-----|-------------|--------------|----------|
| `TELEGRAM__BOT_TOKEN` | str | ✅ | — | Токен Telegram-бота |
| `TELEGRAM__SUPERADMIN_USERS` | list[int] | — | `[]` | ID пользователей с ролью `superadmin` |
| `TELEGRAM__ADMIN_USERS` | list[int] | — | `[]` | ID пользователей с ролью `admin` |
| `TELEGRAM__VIEWER_USERS` | list[int] | — | `[]` | ID пользователей с ролью `viewer` |

> Списки ID принимаются как `[]`, `[1,2]` или `1,2`. Если ID встречается в нескольких списках, действует старшая роль.

## WordPress и БД (`WORDPRESS__*`)

| Переменная | Тип | Обязательно | По умолчанию | Описание |
|------------|-----|-------------|--------------|----------|
| `WORDPRESS__PATH` | path | ✅ | — | Путь к каталогу WordPress |
| `WORDPRESS__DB_HOST` | str | — | `localhost` | Хост БД |
| `WORDPRESS__DB_PORT` | int | — | `3306` | Порт БД |
| `WORDPRESS__DB_NAME` | str | ✅ | — | Имя базы данных |
| `WORDPRESS__DB_USER` | str | ✅ | — | Пользователь БД |
| `WORDPRESS__DB_PASSWORD` | str | ✅ | — | Пароль БД (маскируется в логах) |
| `WORDPRESS__CLI_RUN_AS_USER` | str | — | `www-data` | Пользователь для WP-CLI/распаковки/chown |

## Бэкапы (`BACKUP__*`)

| Переменная | Тип | Обязательно | По умолчанию | Описание |
|------------|-----|-------------|--------------|----------|
| `BACKUP__SITE_NAME` | str | ✅ | — | Имя сайта (используется в имени архива) |
| `BACKUP__PATH_DIR` | path | ✅ | — | Каталог хранения бэкапов |
| `BACKUP__TMP_PATH` | path | ✅ | — | Временный каталог для staging |
| `BACKUP__TIMEZONE` | str | — | `Asia/Almaty` | Таймзона (для меток и планировщика) |
| `BACKUP__MIN_FREE_SPACE_GB` | float | ✅ | — | Минимум свободного места (ГБ), `> 0` |
| `BACKUP__FILE_PREFIX` | str | — | `""` | Префикс имени архива (по умолчанию = `SITE_NAME`) |

## Локальная среда (`CORE__*`)

| Переменная | Тип | Обязательно | По умолчанию | Описание |
|------------|-----|-------------|--------------|----------|
| `CORE__SQLITE_PATH` | path | — | `bot.sqlite3` | Путь к файлу SQLite |
| `CORE__LOGS_DIR` | path | — | `logs` | Каталог логов |
| `CORE__SSH` | str | — | `user@server` | `user@host` сервера для примеров команд `scp`/`rsync` в подсказке `/restore_path` |

## График диска (`DISK_CHART__*`)

| Переменная | Тип | Обязательно | По умолчанию | Описание |
|------------|-----|-------------|--------------|----------|
| `DISK_CHART__ENABLED` | bool | — | `true` | Включить построение графика |
| `DISK_CHART__FORMAT` | str | — | `png` | Формат графика (поддерживается только `png`) |

## Планировщик (`SCHEDULE__*`)

| Переменная | Тип | Обязательно | По умолчанию | Описание |
|------------|-----|-------------|--------------|----------|
| `SCHEDULE__ENABLED` | bool | — | `false` | Включить плановый бэкап |
| `SCHEDULE__BACKUP_CRON` | str | — | `0 3 * * 5` | Cron-выражение планового бэкапа |

## Пути к системным утилитам

Можно задавать как «плоские» переменные (см. ниже) — они переносятся в группу `tools` при старте. Допустима как голая команда (ищется в `PATH`), так и абсолютный путь.

| Переменная | Тип | Обязательно | По умолчанию | Описание |
|------------|-----|-------------|--------------|----------|
| `MYSQLDUMP_PATH` | str | — | `mysqldump` | Путь к `mysqldump` |
| `MYSQL_PATH` | str | — | `mysql` | Путь к `mysql` |
| `TAR_PATH` | str | — | `tar` | Путь к `tar` |
| `ZSTD_PATH` | str | — | `zstd` | Путь к `zstd` |
| `DF_PATH` | str | — | `df` | Путь к `df` |
| `DU_PATH` | str | — | `du` | Путь к `du` |
| `CP_PATH` | str | — | `cp` | Путь к `cp` |
| `WP_CLI_PATH` | str | — | `wp` | Путь к WP-CLI |
| `RUNUSER_PATH` | str | — | `runuser` | Путь к `runuser` |

> `chown` при восстановлении вызывается напрямую и не настраивается отдельной переменной.
