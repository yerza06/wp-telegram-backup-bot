# Задача 07. Мониторинг диска и график

## Цель

Реализовать команды `/disk` и `/disk_chart`, а также общий сервис проверки свободного места для backup/restore.

## Объем работ

- Реализовать безопасный вызов `df` для раздела, где находится целевой путь.
- Реализовать безопасный вызов `du` для:
  - `WORDPRESS__PATH`;
  - `BACKUP__PATH_DIR`.
- Команда `/disk` должна показывать:
  - общий размер раздела;
  - занято;
  - свободно;
  - процент использования;
  - путь проверки.
- Команда `/disk_chart` должна:
  - получить данные `df`;
  - получить размеры `WORDPRESS__PATH` и `BACKUP__PATH_DIR`;
  - построить PNG через `matplotlib`;
  - отправить изображение в Telegram;
  - записать операцию `disk_check` в локальную БД.
- Круговая диаграмма должна показывать:
  - сервер без учета `WORDPRESS__PATH` и `BACKUP__PATH_DIR`;
  - WordPress;
  - бэкапы;
  - свободное место.
- Добавить reusable check для `BACKUP__MIN_FREE_SPACE_GB`.

## Ожидаемые файлы

- `bot/services/disk.py`
- `bot/services/disk_chart.py`
- `bot/handlers/disk.py`

## Критерии готовности

- `/disk` доступна ролям `viewer`, `admin`, `superadmin`.
- `/disk_chart` отправляет PNG.
- Ошибка `df`, `du` или `matplotlib` корректно логируется и возвращает безопасное сообщение.
- Сервис backup использует ту же проверку свободного места.

## Проверки

- Тест парсинга результата `df`.
- Тест парсинга результата `du`.
- Тест построения PNG во временный файл.
