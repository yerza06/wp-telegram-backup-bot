# Задача 11. Планировщик автоматических бэкапов

## Цель

Добавить автоматическое создание полного бэкапа по расписанию через APScheduler.

## Объем работ

- Поддержать настройки:
  - `SCHEDULE__ENABLED`;
  - `SCHEDULE__BACKUP_CRON`;
  - `BACKUP__TIMEZONE`.
- Расписание по умолчанию: `0 3 * * 5`.
- Использовать тот же сервис полного бэкапа, что и `/backup`.
- При активном `backup` или `restore`:
  - не запускать параллельную операцию;
  - записать событие в БД и логи;
  - отправить предупреждение администраторам;
  - отложить плановый запуск до завершения текущей тяжелой операции.
- После планового бэкапа отправить уведомление администраторам о результате.
- Не добавлять Telegram-команды для управления расписанием.

## Ожидаемые файлы

- `bot/services/scheduler.py`
- `bot/services/admin_notifications.py`
- `bot/__main__.py`

## Критерии готовности

- Планировщик стартует только при `SCHEDULE__ENABLED=true`.
- Cron-выражение берется только из конфигурации.
- Плановый backup использует общий lock тяжелых операций.
- Администраторы получают предупреждение об отложенном запуске.
- Telegram не содержит команд изменения расписания.

## Проверки

- Тест парсинга cron-выражения.
- Тест поведения при активной тяжелой операции.
- Ручная проверка с коротким тестовым cron.
