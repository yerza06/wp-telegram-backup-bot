# Задача 05. Права Telegram-пользователей

## Цель

Настроить роли, авторизацию и проверку прав пользователей Telegram без команд управления пользователями внутри бота.

## Объем работ

- Реализовать роли:
  - `superadmin`;
  - `admin`;
  - `viewer`.
- Брать Telegram ID только из переменных:
  - `TELEGRAM__SUPERADMIN_USERS`;
  - `TELEGRAM__ADMIN_USERS`;
  - `TELEGRAM__VIEWER_USERS`.
- Если Telegram ID указан в нескольких списках, выбирать максимальную роль:
  - `superadmin` выше `admin`;
  - `admin` выше `viewer`.
- Добавить middleware или decorator для проверки доступа к handler-ам.
- Для неавторизованных пользователей:
  - не выполнять команды;
  - отправлять короткий отказ на русском языке;
  - писать событие в лог.
- Для пользователей без нужной роли:
  - не выполнять команду;
  - отправлять сообщение о недостатке прав на русском языке;
  - писать событие в лог.
- Описать матрицу доступа:
  - `superadmin`: все команды;
  - `admin`: `/backup`, `/status`, `/disk`, `/disk_chart`, `/backups`, `/clear_cache`;
  - `viewer`: `/status`, `/disk`, `/disk_chart`, `/backups`.

## Ожидаемые файлы

- `bot/core/auth.py`
- `bot/core/roles.py`
- `bot/core/security.py`
- `bot/handlers/`
- `tests/`

## Критерии готовности

- Бот берет пользователей только из `.env` или переменных окружения.
- Нет Telegram-команд для добавления, удаления или изменения пользователей.
- Роль пользователя определяется детерминированно.
- Неавторизованные пользователи не запускают бизнес-операции.
- Пользователи с недостаточной ролью не запускают запрещенные команды.

## Проверки

- Юнит-тесты на парсинг ролей.
- Юнит-тесты на приоритет ролей.
- Тесты авторизации handlers или middleware.
- Ручная проверка через Telegram test bot.
