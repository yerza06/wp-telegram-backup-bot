# Задача 10. Логирование, ошибки, документация и приемка

## Цель

Довести проект до приемочного состояния: надежное логирование, единая обработка ошибок, документация установки и systemd.

## Объем работ

- Настроить локальные логи, например `logs/bot.log`.
- Логировать:
  - запуск и остановку;
  - входящие команды;
  - отказ в доступе;
  - запуск и завершение операций;
  - запуск и отложение плановых операций;
  - ошибки дочерних процессов;
  - stderr системных команд;
  - результаты очистки кэша.
- Ввести единые exception-типы для доменных ошибок:
  - отсутствующая утилита;
  - неверный `WORDPRESS__PATH`;
  - недоступный `BACKUP__PATH_DIR`;
  - нехватка места;
  - ошибка `mysqldump`;
  - ошибка `mysql`;
  - поврежденный архив;
  - активная тяжелая операция;
  - ошибка WP-CLI;
  - ошибка построения графика.
- Добавить безопасное преобразование ошибок в Telegram-сообщения.
- Обновить README:
  - установка зависимостей;
  - пример `.env`;
  - запуск локально;
  - установка systemd service;
  - базовая диагностика.
- Добавить приемочный чеклист по `docs/technical-spec.md`.

## Ожидаемые файлы

- `bot/core/logging.py`
- `bot/core/errors.py`
- `bot/utils/sanitize.py`
- `README.md`
- `docs/acceptance-checklist.md`
- `deploy/telegram-wp-backup.service.example`

## Критерии готовности

- Подробные ошибки остаются в локальном логе.
- Telegram получает короткие безопасные сообщения.
- Логи не содержат секреты.
- Есть инструкция запуска через `systemd`.
- Все критерии приемки из ТЗ перенесены в checklist и проверены.

## Проверки

- `python -m compileall bot`
- Тесты sanitizer и domain exceptions.
- Ручная smoke-проверка команд Telegram.
- Ручная проверка systemd unit на тестовом сервере.
